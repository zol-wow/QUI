# Per-locale options-search cache addons — dev AddOns symlinks

The i18n work (plan Task 9) adds 10 load-on-demand, localized options-search
cache addons alongside the existing English `QUI_OptionsSearch`. They are
generated by `tools/i18n/gen_all_caches.sh` (which runs
`tools/generate_search_cache.lua <locale>` once per locale).

This build environment has **no WoW AddOns directory and no committed symlink
script**, so the symlinks must be created manually on a dev machine. The suite
split (`[[suite-split-landed-working-tree]]`) symlinks each `QUI_*` repo dir into
`<WoW>/_retail_/Interface/AddOns/`; do the same for these 10 siblings.

## Addon dirs that need symlinking

```
QUI_OptionsSearch_deDE
QUI_OptionsSearch_esES
QUI_OptionsSearch_esMX
QUI_OptionsSearch_frFR
QUI_OptionsSearch_itIT
QUI_OptionsSearch_ptBR
QUI_OptionsSearch_ruRU
QUI_OptionsSearch_koKR
QUI_OptionsSearch_zhCN
QUI_OptionsSearch_zhTW
```

## Example (adjust `WOW_ADDONS` to your install)

```bash
WOW_ADDONS="$HOME/Games/World of Warcraft/_retail_/Interface/AddOns"
REPO="$(pwd)"   # run from the QUI repo root
for loc in deDE esES esMX frFR itIT ptBR ruRU koKR zhCN zhTW; do
  ln -snf "$REPO/QUI_OptionsSearch_${loc}" "$WOW_ADDONS/QUI_OptionsSearch_${loc}"
done
```

Each addon is `LoadOnDemand: 1` and is loaded by
`QUI_Options/framework.lua` (`SearchCacheAddonName()` → enUS fallback) the first
time the settings search box is used under a matching client locale. The in-repo
cache dirs (`.toc` + `search_cache.lua`) are what ships in the release artifact;
the symlinks only matter for live dev testing.
