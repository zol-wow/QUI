## Findings considered and rejected (so nobody re-audits them)

- **XP datatext nil guards** (`QUI_Datatexts/datatexts/datatexts.lua:3578-3640`):
  `UnitLevel("player")`/`UnitXP("player")` returning nil is theoretical for
  the player unit; the division already guards `maxXP == 0`. Not a bug in
  practice.
- **Shared-ticker pairs() mutation** (`datatexts.lua:44-63`): assigning nil
  to an *existing* key during `pairs` is legal Lua; `Unregister` only nils
  existing keys. Not a bug.
- **Chat editbox OnEvent lacks pcall** (`QUI_Chat/chat/editbox_basics.lua:479`):
  handler is guarded (`if not tabIndex then return end`, `if TabUI and …`);
  a Lua error would not unregister the frame's events anyway. Claimed impact
  was wrong.
- **Profile import DoS** (`core/profile_io.lua`): validation enforces
  `MAX_IMPORT_DEPTH=20`, `MAX_IMPORT_NODES`, cycle detection via visited set,
  and key-type checks — traversal is bounded. Also verified: no
  `loadstring`/`RunScript`/`setfenv` anywhere in the import pipeline (the
  catastrophic risk class for shared import strings is absent).
- **package.sh sed "command injection"**: macOS BSD sed has no GNU `e`
  command; worst case is malformed staged files from a weird alias the
  maintainer typed themselves. Robustness nit, not security.
- **release.sh honors `$EDITOR`**: standard platform convention (by-design
  per the audit playbook), maintainer-local script.
- **release.yml secret masking**: GitHub Actions masks registered secrets by
  default; explicit `add-mask` would be defense-in-depth only.
- **"DamageMeter has zero tests"**: ~20 test files match damage/meter;
  `docs/features/damage-meter.md` exists. Mis-measured.
- **"Migration chains untested"**: `tools/test_profiles.lua` +
  `tests/fixtures/{legacy,edge,current}` are end-to-end chain tests
  (v22→, v31→, v32→, v34→, v40→41), plus unit tests through v45
  (`chat_window_position_single_store_migration_test.lua`). Coverage is
  good; ship a fixture with each new migration as standing practice.
- **"Granular profile export hidden"**: `ExportProfileSelectionToString` has
  a UI consumer (`core/settings/content/import_export_content.lua:261`).
- **"Release zips/npm-debug.log committed"**: refuted — `git ls-files` shows
  they are untracked local files. (Optional hygiene: add `*.zip`,
  `npm-debug.log` to `.gitignore`; not worth a plan.)
- **Chat message-store compaction churn** (`QUI_Chat/chat/message_store.lua:29`):
  documented, deliberate amortized-trim design ("Keeps Append O(1) during
  spam bursts"); a ring buffer adds wrap-around risk to a secret-safety-
  critical file for negligible GC gain.
- **CDM fallback visibility-table allocation**
  (`QUI_CDM/cdm/cdm_icon_renderer.lua:2814`): fallback path effectively
  never runs (policy initialized at load); per the finder's own analysis.
- **Vendored-lib findings (LibOpenRaid validation, Ace3 pinning,
  CallbackHandler coupling)**: repo convention is "don't edit vendored
  libs"; operational risks noted, no action warranted now.
- **Migrations accumulation (27 functions, v0–v45)**: not a performance
  problem at this scale; revisit policy if the count doubles.
